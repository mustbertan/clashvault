Tech Blueprint
Recommended Tech Stack
Full-stack PWA approach — fast, SEO-friendly, deployable on Vercel/Railway.
Next.js 14 (App Router) Supabase (PostgreSQL) Tailwind CSS Cloudinary (Images) Vercel (Deploy)
Database Schema
-- LAYOUTS TABLE layouts { id uuid PRIMARY KEY title varchar(120) th_level int (6–16) village enum (home | builder) type enum (war|farm|trophy|hybrid|progress) image_url text coc_link text -- hidden from public API uses int DEFAULT 0 favs int DEFAULT 0 created_at timestamp is_active bool DEFAULT true added_by uuid → users } -- FAVORITES TABLE user_favorites { id uuid PRIMARY KEY user_id varchar (anon fingerprint) layout_id uuid → layouts created_at timestamp UNIQUE(user_id, layout_id) }
Sharing Link Logic
The public share URL never exposes the CoC link. The CoC link is only served via a server-side API call that also increments the uses counter.
Share URL: myapp.com/layout/[id] API Route: POST /api/layout/[id]/use → increments uses counter → returns { cocLink } (one-time) Result: share link = your app page CoC link = gated behind counter API
Favorites System
-- No login required (anon fingerprint) 1. Generate userId = localStorage UUID 2. POST /api/layout/[id]/favorite { userId } 3. Upsert user_favorites table 4. Update layouts.favs count 5. Return { favorited: bool, total: int }
Admin Panel
Simple password-protected route at /admin. Auth via Supabase Auth (email/password). The admin form auto-generates the public layout page on submit.
Route: /admin/layouts Upload image → Cloudinary Paste CoC link Auto-creates page
PWA Config
// next.config.js withPWA({ dest: 'public', register: true, skipWaiting: true }) // manifest.json { "name": "ClashVault", "display": "standalone", "theme_color": "#F5A623", "background_color": "#1A0F00" }
Page Structure
app/ ├── page.tsx → Home feed ├── layout/ │ └── [id]/page.tsx → Base detail (SEO) ├── browse/page.tsx → Filter + search ├── favorites/page.tsx → Saved bases └── admin/ ├── login/page.tsx └── layouts/ ├── page.tsx → Layout list ├── new/page.tsx → Add layout └── [id]/edit → Edit layout
API Routes
GET /api/layouts → list (filter params) GET /api/layouts/[id] → single layout POST /api/layouts/[id]/use → incr uses + cocLink POST /api/layouts/[id]/fav → toggle favorite POST /admin/api/layouts → create (auth) PUT /admin/api/layouts/[id] → edit (auth) DEL /admin/api/layouts/[id] → delete (auth)
